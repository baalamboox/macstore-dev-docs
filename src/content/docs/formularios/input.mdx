Componentes personalizados para la implementacion de formularios
Dentro del proyecto se utilizan dos tipos de input text,  el primero  se usa principalmente en el proyecto de CX
y el segundo es el input normal que se usa para formularios para el administrativo

<Aside>Para que los input de cx funcionen en diferentes navegadores es necesario agregar un id al input y que este sea el mismo que el for del label.  </Aside>

## Input Email

<div class="inputs">
  <Card title="input Email" icon="setting">
   <InputEmail/>

<div class="tabb-m">
<Tabs>
<TabItem label="HTML">
 ```html
<div class="entryarea w-100">
    <input id="nws3" class="inputani" type="email"  required />
    <label  id="labelnws3" class="labelline" for="nws3">Confirmar correo</label>
</div>
 ```
</TabItem>
<TabItem label="JS">
```JavaScript  
 const textInput3 = document.getElementById('nws3');
 textInput3.addEventListener('input', function () {
     const inputValue2 = textInput3.value.trim();
     var labelinput2 = document.getElementById("labelnws3");
     if (inputValue2 !== '') {
 
         labelinput2.classList.add('focusclassinputemail');
     } else {
 
         labelinput2.classList.remove('focusclassinputemail');
     }
 }); 
 ```
</TabItem>
 </Tabs>
</div>




<Aside>Para su correcta funcionalidad, es importante agregar el js referenciando el id del input y el label de forma correspondiente cada que se duplique este componente  </Aside>
  </Card>

</div>
